## Root cause and fix summary
Root cause: CodeGraph dist wrappers bundled `jsonc-parser` through its UMD entry, which left runtime `require("./impl/format")` and sibling `./impl/*` requires inside single-file `dist/serve.js` and `dist/cli.js`. The installed Codex plugin ships those wrappers without `jsonc-parser/lib/umd/impl`, so Node crashed before wrapper activation.
Fix: `packages/utils/src/jsonc-parser.ts` now imports `jsonc-parser/lib/esm/main.js`, causing Bun to embed the parser implementation directly. Regenerated CodeGraph `dist/serve.js` and `dist/cli.js` no longer contain `./impl/format`; both local and installed wrappers invoke fake CodeGraph as `serve --mcp`.
Residual note: the first evidence-captured `codex-qa install-verify` overlapped with another install build and hit an ENOTEMPTY cleanup race in generated skills; the serial rerun immediately below it passed.
